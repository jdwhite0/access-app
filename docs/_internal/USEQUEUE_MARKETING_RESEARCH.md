# INTERNAL — design reference, not user-facing in app UI

**Queue (usequeue.com) marketing research**  
**Date:** 2026-06-03  
**Purpose:** Inform ACCESS public homepage copy/design pass — extract communication mechanics, not visual clone.  
**Sources:** https://usequeue.com (homepage), https://usequeue.com/pricing, live browser review (hero + page structure).  
**Lens:** Universal audience starting a business with AI help; ACCESS brand = white, navy, cyan, calm, premium (not cyberpunk).

---

## Executive summary

Queue sells to **productized-service agencies** with a single-scroll homepage that stacks **outcome headline → category definition → risk-free signup → third-party trust → repeated “chapter” sections** (each with H2 benefit, one-sentence job-to-be-done, then **tabs + real UI screenshots** or **3-up outcome columns**). Copy is **plain business English** with occasional punchy specificity ($0.00 fees, 15 seconds, 2 clicks). Visuals mix **premium abstract 3D** (hero) with **literal product UI** (mid-page) so visitors learn *what it does*, not just that it looks expensive.

ACCESS already has warmer hero copy and a calm brand direction; the gap is **proof, product literacy, and narrative section rhythm** — not more adjectives.

## 1. Homepage hero

### Structure

| Element | Queue pattern |
|--------|----------------|
| **H1** | Short outcome + consolidation: *“Run your agency with 1 platform”* — verb + audience + “one” metaphor |
| **Subhead** | Category claim + capability triad: *“operating system for agencies”* + billing, projects, feedback + quality cue (*“beautifully designed”*) |
| **Primary CTA** | Action + price signal: *“Create free account”* (solid accent button) |
| **Secondary CTA** | Self-serve education: *“Video Guides”* (muted button, same row) |
| **Microcopy under CTAs** | Risk reversal + social proof in one line: *“Free until your first client — Trusted by thousands of agencies”* |
| **Trust row** | G2 “Users Love Us” / 5 Stars / **Backed by Y Combinator** (clickable badges) |
| **Audience line** | Breadth without jargon: *“Trusted around the world from freelancers to large enterprises”* |

### Layout (visual)

- **Dark charcoal** full-bleed background; white headline with subtle gradient on second line of H1 (“with 1 platform”).
- **Split hero:** copy left (~55%), **large iridescent 3D torus** right — no product UI in fold; signals “premium / engineered” before proof.
- Minimal nav: wordmark + hamburger only.

### Tone

- **Confident operator**, not hype-bro. Speaks to agency owners who already run client work.
- Assumes pain = **tool sprawl** (billing + projects + feedback scattered).
- **Not** developer-first; no API/code in hero.

### ACCESS translation (hero)

- Keep ACCESS warmth; borrow **structure**: H1 = life outcome + “one place”; subhead = **3 plain verbs** (plan / remember / get help) not feature names first.
- Add **risk line under CTAs** when true (e.g. founding pricing, no card, what “free” includes).
- Replace self-only trust claims with **one external proof slot** when available (even “Built by JD AI Systems” + status link beats three identical self-assertions).

## 2. Visual system

### Hero layer

- **Abstract 3D** (metallic torus, pink/cyan reflections) = emotional premium, not feature explanation.
- Works because subhead immediately names **jobs**; art does not carry meaning alone.

### Mid-page layer

- **Full-width UI screenshots** inside light/white sections (client portal, paywalls, kanban, feedback on websites/videos).
- **Tabs** swap screenshot + micro-headline (Client Portal: white-label, requests, subscriptions, onboarding).
- **Icon triads** for money features: Create Invoices / Paywall Delivery / No Fees — each with **one concrete sentence** and implied UI above.

### Social proof visuals

- **Logo carousel** with instruction: *“Click on the logo. They're real customers.”* — interactive proof, not static strip.
- **Use-case cards** (e.g. Video Agencies / TubeWorks) with category + short story (carousel; some placeholder “Accounting Firms” lorem visible in export — treat as CMS noise).

### Integrations

- Recognizable app logos (Google Docs, Airtable, Calendly, Stripe, PayPal) — **borrowed trust** from tools audience already uses.

### Imagery rule

> Every visual either **shows the workflow** or **borrows trust from a known brand**. Decoration without caption is rare.

### ACCESS translation (visual)

- **Light ACCESS world:** white/navy/cyan; hero art can stay abstract/calm (orb, soft geometry) but **first scroll below fold** should show **labeled product frames** (Home + JYSON thread, memory snippet) with real copy inside mocks — not gray placeholder lines only.
- One **tabbed “see how it works”** block beats five identical chrome-only panels.

## 3. Section rhythm

### Macro pattern (top → bottom)

1. Hero + trust badges  
2. Logo strip + **“Trusted by 1,500+ businesses”** + use-case carousel  
3. **Payments chapter** — H2 implied via feature labels; 3 columns with outcomes  
4. **Client Portal** — emotional H2 (*“they'll love using”*) + paragraph + **tabs + screenshot**  
5. **Integrations** — “Bring your favorite apps” + logo grid + time claim  
6. **Monetization** — sell subscriptions/one-offs/credits + tabs  
7. **Reliability** — 99.99% uptime + security (enterprise nod)  
8. **Feedback** — pain opener (*“No more confusing Slack messages”*) + Websites / Videos / Images tabs  
9. **Project board** — collaboration + 4 sub-features (private comments, forms, control, Slack)  
10. **Mobile app** — notifications on the go  
11. **Blog** — growth content  
12. **Testimonials** — *“Meet Queue users”* long quotes, names, companies, migration stories  
13. Footer — product taxonomy + register/login  

### Chunking mechanics

- **One H2 = one chapter job** (not “Features” dump).
- Subheads are **6–12 words**, often second person or client-benefit (*“they'll love”*).
- Bullets use **H6 titles + one line** under tabs (scannable).
- **Carousels** for logos, use cases, testimonials — density without infinite scroll fatigue.

### Trust / proof ladder

1. YC + G2 (instant credibility)  
2. Count (*1,500+ businesses*) + vertical examples (Webflow agencies, studios)  
3. **Founder-grade testimonials** (Trello + Stripe migration, support fixed bug in 2 minutes)  
4. **Pricing page** doubles down: *“Free until you get a client”* + empathetic *“Starting out is hard. We've been there.”*

### Feature explanation

- **Pain → mechanism → outcome** (feedback section is the clearest).
- **Numbers as copy devices:** $0.00 fees, 99.99%, 15 seconds, 2 clicks, 2 seats included.

### ACCESS translation (rhythm)

Suggested ACCESS chapter order (calm, universal):

1. Hero + proof microline  
2. **“Stop restarting every conversation”** — memory / continuity (pain → ACCESS)  
3. Tabbed product walkthrough (Plan → JYSON → Projects → Memory)  
4. **“One place, not five apps”** — consolidation (mirror Queue’s triad)  
5. Plans with **founding story line** (Queue’s “free until first client” equivalent)  
6. Testimonials or founder quote — when real users exist  
7. Help / status + final CTA

## 4. Language

### Register

- **~80% everyday business**, **~20% product nouns** (white-labeled, subscription, paywall, embed).
- Avoids engineering slang in hero; saves it for Platform/Security footer links.

### Phrases that work (adapt, don’t copy)

| Queue phrase | Why it works |
|--------------|--------------|
| *“Run your agency with 1 platform”* | Verb + audience + singularity |
| *“operating system for agencies”* | Category frame (ACCESS: *“home base for your business”* not “OS”) |
| *“brings X, Y, and Z together”* | Triad reduces perceived complexity |
| *“Free until your first client”* | Specific risk reversal |
| *“Client Portal they'll love using”* | Emotional benefit in H2 |
| *“No more confusing slack messages or email chains”* | Named pain, familiar tools |
| *“Keep 100% of what you make with $0.00 fees”* | Concrete economics |
| *“Most embeds takes less than 15 seconds”* | Time proof lowers setup fear |
| *“Integrated with 2 clicks”* | Friction math |
| *“Click on the logo. They're real customers.”* | Human, slightly playful proof |
| *“It's like having a partner in business... not just another software vendor”* | Testimonial reframes relationship |

### Pricing page tone

- **Empathy first:** *“Starting out is hard. We've been there.”*
- Tier names: Starter / Growth / Enterprise — limits listed as **facts**, not marketing fluff.

## 5. What makes the communication “amazing” — mechanics ACCESS can adopt

1. **Singularity headline** — One platform / one place / one story; number optional but memorable.  
2. **Category + triad subhead** — Define what ACCESS is in one sentence with three plain outcomes.  
3. **Dual CTA hierarchy** — Primary = start; secondary = learn (video, tour, or “see how it works”).  
4. **Risk reversal directly under buttons** — Specific condition (free until X, founding price, no card).  
5. **Third-party trust in the fold** — Badges, counts, or status — not only self-described trust strips.  
6. **Chapter-based scroll** — Each section answers one question: *Who is it for? What do I get? How hard is setup? Is it safe? Who else uses it?*  
7. **Tabs + real UI** — Teach the product; swap one screenshot per job story.  
8. **Pain-named copy** — “Scattered notes,” “explaining yourself again,” “five apps” before feature names.  
9. **Micro-specificity** — Times, dollars, counts where honest (even “3 steps to first plan” if true).  
10. **Testimonials as migration stories** — Prior tool → ACCESS outcome; support humanity beats “Goated 🐐” tone for ACCESS audience.

## 6. What NOT to copy for ACCESS

| Queue element | Why wrong for ACCESS |
|---------------|----------------------|
| **Dark mode + iridescent 3D** | Conflicts with white / navy / cyan calm premium |
| **“Operating system” / agency-only positioning** | Feels technical or B2B-agency; mom / universal starter may bounce |
| **Agency / client-portal / white-label narrative** | ACCESS is **founder’s space + AI guide**, not client-facing agency OS |
| **Paywall, invoices, $0 platform fees** | Different product truth; misleading if echoed |
| **YC / G2 badge wall without equivalents** | Empty authority if ACCESS can’t substantiate |
| **Twitter-style quotes** (*“Queue pilled,” “Goated”*) | Wrong voice for warm, universal brand |
| **Long dark-scroll Webflow density** | ACCESS should breathe; fewer carousels, more clarity |
| **Placeholder / duplicate carousel cards** | Signals unfinished; ACCESS must ship tight copy only |
| **“Beautifully designed platform” filler** | Generic; ACCESS should cite **felt outcomes** (calm, remembers you, next step clear) |
| **Footer product sprawl** | 10+ SEO links — confuses single-product ACCESS story |

## 7. Pricing page notes (subpage)

- URL: `/pricing` — headline *“Free until you get a client”* with empathetic subcopy.
- **Starter FREE** until first client; explicit platform limits (2 projects, 2 seats, storage, uploads).
- **Growth $49/mo** “Most Popular” — anchors mid-tier.
- **Enterprise** contact — SOC2, multi-workspace, dedicated success.
- FAQ answers **objections** (how free works, security, fees, file types, pause/unpause).
- Reuses testimonial carousel from homepage.

**ACCESS takeaway:** Pair plans with **one sentence of empathy + one clear “start here” tier**; FAQ for “what is ACCESS / is my data private / can I cancel” not agency billing mechanics.

## 8. ACCESS `PublicHomePage.tsx` — three biggest gaps vs Queue

| # | Gap | Queue | ACCESS today |
|---|-----|-------|----------------|
| **1** | **External proof & specificity** | 1,500+ businesses, G2/YC, logo carousel, long user quotes | `TrustStrip` only states ACCESS benefits — no names, numbers, or third-party validation |
| **2** | **Product literacy in UI** | Tabbed sections with real screenshots per workflow | `PRODUCT_PANELS` are abstract chrome + gray lines — visitor cannot see *what happens* in JYSON/memory |
| **3** | **Pain → chapter narrative** | Each H2 solves one job; pain lines (“no more Slack…”) before features | Capability cards are feature-first (“Memory that sticks”) with no prior pain, no tabbed depth, shallow scroll story before plans |

### Secondary gaps (honorable mention)

- No **risk-reversal microcopy** under hero CTAs (Queue: free until first client).
- No **secondary learn CTA** (Video Guides / product tour).
- Plans section lacks **empathy framing** and objection FAQ block.

## 9. Recommended ACCESS homepage pass (copy/design only)

### Hero

- **Eyebrow:** keep universal starter framing.
- **H1 option:** *“Run your idea from one calm place”* (structure borrow; voice ACCESS).
- **Subhead triad:** plan your next step · save what matters · AI that remembers you — **no jargon first**.
- Under CTAs: founding offer line + link to status or privacy as trust substitute until testimonials exist.

### Below fold

1. Replace or augment `TrustStrip` with **one proof module** (founding user quote, JD Productions use case, or “Private by default” + link).  
2. Add **tabbed product section** with 2–4 real static screenshots (even marketing captures from staging).  
3. Insert **pain band** before capability cards: *“Starting over every time you open a new app?”* → then three cards.  
4. Plans: one empathetic line above grid; optional 3-question FAQ accordion.  
5. Final CTA: repeat primary only; secondary = contact or tour.

### Visual

- Stay **light**; hero artwork can remain abstract if labeled product UI follows within one scroll.
- Cyan accent on **one** primary button; navy for text hierarchy.
